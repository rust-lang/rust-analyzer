update_fallback⚠
“Fallback” for maybe-update that is suitable for fully owned T that implement Eq. In this version, we update only if the new value is not Eq to the old one. Note that given Eq impls that are not just structurally comparing fields, this may cause us not to update even if the value has changed (presumably because this change is not semantically significant).
