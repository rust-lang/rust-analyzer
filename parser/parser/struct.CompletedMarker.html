pub(crate) fn precede(self, p: &mut Parser<'_>) -> Marker

This method allows to create a new node which starts before the current one. That is, parser could start node A, then complete it, and then after parsing the whole A, decide that it should have started some node B before starting A. precede allows to do exactly that. See also docs about Event::Start::forward_parent.

Given completed events [START, FINISH] and its corresponding CompletedMarker(pos: 0, _). Append a new START events as [START, FINISH, NEWSTART], then mark NEWSTART as START’s parent with saving its relative distance to NEWSTART into forward_parent(=2 in this case);
